## 1. 项目概述

AI气象预测数据中心与可视化平台是一套面向气象数据管理与行业应用的可弹性扩展架构体系。平台以 **ghr_9km** 综合气象模型为核心，实现 PB 级气象数据的统一接入、长期归档、快速检索、批量导出与标准化 API 服务；同时提供极端天气预警全流程能力，以及物流、粮食、电力等行业场景的可视化与定制服务。

平台采用 **React + TypeScript + Vite** 构建单页应用，当前版本以平台示例数据驱动全功能演示，生产环境可对接真实后端 API。

## 2. 模块一：综合气象模型服务

### 2.1 功能说明

提供以 **ghr_9km** 为核心的综合气象预报查询能力，覆盖从轻量化单点查询到专业多变量、批量多点、行政区域及历史回溯的全谱系数据获取场景。支持 23 种标准气象变量及 13 个高空气压层，满足业务系统对精细化预报数据的统一访问需求。

**核心能力：**
- 简易气象查询：单点 11 天逐小时预报，零门槛操作
- 专业气象查询：多变量（最多 4 个）、多气压层、UTC 起报时次选择
- 批量与区域查询：多点批量（≤100 点）、行政区域、10 年历史回溯
- 模型参数展示：模型名称、起报时间、预报时次、覆盖范围校验

### 2.2 技术实现

| 实现项 | 说明 |
|--------|------|
| 页面组件 | `SimpleQueryPage.tsx`、`ProfessionalQueryPage.tsx`、`BatchQueryPage.tsx` |
| 图表渲染 | `WeatherCharts.tsx`（温度/辐射/风速/降水/多变量折线图） |
| 地图交互 | `WeatherMap.tsx`（单点标记、点击选点） |
| 变量选择 | `VariableSelector.tsx`（地表 + 气压层，数量上限控制） |
| 数据生成 | 简易/专业查询预报数据生成接口 |
| 状态管理 | Zustand 存储查询结果与 loading 状态 |
| 范围校验 | 经度 72°–137°E、纬度 17°–55°N 前端校验 |

**ghr_9km 模型参数：**

| 参数 | 值 |
|------|-----|
| 空间分辨率 | 9 km |
| 预报时长 | 264 小时（11 天） |
| 起报频次 | UTC 00/06/12/18 时 |
| 支持变量 | 23 种（含 13 气压层衍生变量） |

### 2.3 对应页面/路由

| 路由 | 页面名称 | 功能摘要 |
|------|----------|----------|
| `/simple-query` | 简易气象查询 | 城市/坐标/地图选点，4 类预报图表，JSON 导出 |
| `/professional-query` | 专业气象查询 | 多变量曲线、原始数据表格、UTC 时次、数据导出 |
| `/batch-query` | 批量与区域查询 | 多点/区域/历史三种模式，NetCDF/CSV 导出 |

### 2.4 数据来源说明

| 数据项 | 来源 | 说明 |
|--------|------|------|
| 简易查询结果 | 平台预报数据接口 | 264 小时逐小时温度/辐射/风速/降水，含日变化规律 |
| 专业查询结果 | 专业查询数据接口 | 按选定变量过滤气压层数据输出 |
| 批量查询坐标 | 批量查询坐标集 | 6 个预置经纬度坐标 |
| 历史回溯数据 | 历史年景数据集 | 2016–2025 年均温/降水/最大风速/预警次数 |
| 模型标识 | 固定 `'ghr_9km'` | 所有查询结果统一标注模型名称 |

## 3. 模块二：数据通道构建

### 3.1 功能说明

构建从多源数据接入、标准化清洗加工到 PB 级长期存储的完整数据通道，并通过标准化 API 对外提供气象与预警数据服务。支持 ERA5、ECMWF HRES、官方预警、行业业务数据等多源异构数据的统一纳管。

**核心能力：**
- 多源数据接入管道（FTP/HTTP/REST API/消息队列/文件上传）
- 数据清洗（缺失值处理、异常值修正、预警去重）
- 数据加工（格式转换、分辨率调整、时空聚合）
- PB 级冷热分层存储与生命周期管理
- OpenAPI 3.0 规范接口文档

### 3.2 技术实现

| 实现项 | 说明 |
|--------|------|
| 接入管理页 | `DataProcessingPage.tsx` — 管道列表、清洗统计、手动触发清洗 |
| 存储管理页 | `DataStoragePage.tsx` — 分层策略、数据集列表、完整性指标 |
| API 文档页 | `APIDocsPage.tsx` — POST 单点查询、GET 历史任务列表 |
| 运行概览 | `DashboardPage.tsx` — 多源接入状态表、系统对接状态 |
| 常量配置 | `constants.ts` — `DATA_LIFECYCLE_CONFIG` 冷热分层参数 |
| 接入管道 | 接入管道数据集 — 5 条接入管道 |
| 数据源 | 数据源清单 — 6 个数据源（含行业数据） |
| 数据集 | 数据集清单 — 5 个 PB 级数据集 |

**接入管道类型：**

| 管道 | 接入方式 | 格式 | 延迟目标 |
|------|----------|------|----------|
| ERA5 定时同步 | FTP | NetCDF | ≤5 min |
| ECMWF HRES 实时推送 | HTTP/HTTPS | GRIB2 | ≤3 min |
| 官方预警数据接入 | REST API | JSON/XML | ≤1 min |
| 物流路径数据导入 | 消息队列 | CSV | 实时 |
| 粮食产区地理数据 | 文件上传 | GeoJSON | 每日 |

### 3.3 对应页面/路由

### 11.2 技术实现

| 实现项 | 说明 |
|--------|------|

| 规则结构 | WarningRule — conditions[] + logic + spatialScope |

| 版本追踪 | 规则 version 字段（v1.0–v2.1） |

**预警规则引擎结构：**

```
WarningRule
├── name / enabled / priority
├── warningType / warningLevel
├── conditions[] — { element, operator, threshold, unit }
├── logic — 'and' | 'or'
├── spatialScope — 'global' | 'region' | 'custom'
├── effectiveMode — 'immediate' | 'scheduled'
└── version / triggerCount / updatedAt
```

**数据模板创建工作流：**

```
Step 1: 基础信息 + 地理范围（矩形/坐标/区域）
    ↓
Step 2: 气象要素选择（VariableSelector 全选/分类）
    ↓
Step 3: 推送配置（时次/间隔/格式/空间处理）
    ↓
保存模板 → 可选开启定时推送
```

### 11.3 对应页面/路由
